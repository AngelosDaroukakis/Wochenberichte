Processing,p5.js

Processing ist eine objektorientierte, stark typisierte Programmiersprache mit zugehöriger integrierter Entwicklungsumgebung. Die Programmiersprache ist auf die Einsatzbereiche Grafik, Simulation und Animation spezialisiert. Processing wird in einem quelloffenen Projekt entwickelt. Processing hat den Charakter einer stark vereinfachten Version der Programmiersprache Java, ermöglicht es, Interaktionen und visuelle Elemente zu programmieren und richtet sich vorwiegend an Gestalter, Künstler und Programmieranfänger. Mit p5.js wurde eine Erweiterung für JavaScript entwickelt, welche die Prinzipien und Funktionen von Processing mit Webtechnologien umsetzt. Processing kombiniert die Programmiersprache mit einer integrierten Entwicklungsumgebung, die speziell für Menschen entwickelt wurde, die in den Bereichen visuelle Kunst, Design und Elektronik arbeiten. Sie wurde entwickelt, um das Schreiben von Code zugänglicher und benutzerfreundlicher zu gestalten. Die Processing IDE kann für eine Vielzahl von Projekten verwendet werden. Sie ist besonders nützlich für die Erstellung von interaktiven Grafiken, Animationen und Spielen. Die Sprache bietet eine einfache Syntax und eine Vielzahl von Funktionen, die das Erstellen von visuellen Projekten erleichtern. In den Editor ist direkt eine Vorschau integriert, in der zwei- und dreidimensionale Grafiken angezeigt werden können. p5.js ist eine kostenlose, quelloffene JavaScript-Bibliothek, die für kreatives Programmieren und die Erstellung interaktiver visueller Kunstwerke, Animationen und Spiele im Webbrowser entwickelt wurde. Sie bietet Werkzeuge und Funktionen, die es Künstlern, Designern und Anfängern ermöglichen, mit Code zu malen und zu gestalten, ähnlich wie sie es in einem Notizbuch tun würden. p5.js macht das Programmieren zugänglich, indem es die Komplexität von JavaScript durch eine einfachere Syntax und eine unterstützende Community reduziert.

GitHub

Auf dem Bild sieht man die Octocat von GitHub. das Bild stammt aus GitHub.

GitHub ist eine Plattform auf Basis des Versionskontrollsystems Git, die es ermöglicht, Änderungen an Dateien und Quellcode nachzuverfolgen und ältere Versionen wiederherzustellen. Sie erweitert Git um eine benutzerfreundliche Oberfläche und bietet zahlreiche Funktionen, die insbesondere die Zusammenarbeit im Team erleichtern.

GitHub ist eine bekannte Plattform, die auf dem Git Versionskontrollsystem basiert. Git wurde ursprünglich entwickelt und erlaubt das Nachverfolgen von Änderungen an Dateien bzw. an Quellcode, das Speichern von Versionen und jederzeit auch das Zurückholen älterer Versionen. Auf diesen Funktionen baut GitHub auf und erweitert Git um ein benutzerfreundliches Interface sowie eine Vielzahl von Funktionen, die Teamarbeit erleichtern. Im Wesen von GitHub speichert man dort seine Projekte in sogenannten Repositories. Ein Repository (oft auch nur Repo genannt) beinhaltet neben dem Quellcode auch Dokumentationen, Bilder, Konfigurationsdateien oder Tests. Repositories können öffentlich oder privat sein. In öffentlichen Repos kann jeder den Quellcode einsehen, herunterladen und Vorschläge zur Verbesserung in Form von sogenannten Pull Requests einreichen. Damit ist GitHub ein zentraler Treffpunkt der Open-Source-Welt. Millionen von Entwickler:innen aus aller Welt pflegen dort ihre Projekte ein, verbessern die Software anderer oder machen sich gemeinsam an neue Ideen. Ein zentrales Konzept sind die Branches. Über diese lässt sich zu einem Projekt parallel ein zweiter, dritter oder zehnter Entwicklungszweig führen, ohne den Hauptzweig zu beeinträchtigen. Sobald etwas fertig ist, kann man über einen Pull Request um die Übernahme in den Hauptbranch bitten. Andere können dann die Änderung überprüfen und bei Gefallen übernehmen. So ist ein transparenter und kontrollierter Prozess entstanden, bei dem Fehler schneller gefunden und behoben werden. Neben dem eigentlichen Code-Hosting haben GitHub zahlreiche Organisations- und Automatisierungsfeatures: Mit Issues können Fehler dokumentiert werden oder Wünsche für neue Funktionen geäußert werden. Wiki-Seiten bieten die Möglichkeit, Anleitungen und Hintergrundwissen zu sammeln. GitHub Actions ermöglichen das Automatisieren von Aufgaben, wie Testläufe, Builds oder Deployments. Project Boards erleichtern die Planung und Organisation von komplexen Entwicklungsprozessen und funktionieren ähnlich wie Kanban-Boards. GitHub nützt nicht nur zahlreichen Firmen, sondern ebenso Einzelpersonen und kleine Teams. Den Dienst nutzen viele Programmierer, um ihre Karriere zu dokumentieren, oder auch um gemeinsam zu lernen. Unternehmen wiederum profitieren von privaten Repositories und der Möglichkeit, GitHub in bestehende Arbeitsprozesse zu integrieren. Seit 2018 gehört GitHub zu Microsoft. Auch wenn es anfänglich einige Skeptiker gab, so blieb GitHub dennoch eine der wichtigsten Plattformen für Entwickler*innen weltweit und hat ihren Status, wenn nicht sogar ihre Bedeutung, weiter ausgebaut. Heute ist GitHub nicht einfach ein Ort, an dem Code abgelegt wird, sondern ein soziales Netzwerk für Entwickler*innen, dass Austausch, Feedback und gemeinsames Lernen ermöglicht. Auf den Punkt gebracht: GitHub vereint Versionskontrolle, Teamarbeit und Community. Es ist das Tool, das Softwareentwicklung transparenter, effizienter und kollaborativer macht – und damit Innovation auf der ganzen Welt ermöglicht.

Fazit

GitHub ist wie eine Bibliothek für Informatiker wiel man dort alles finden kann und auch von anderen die sachen übernehmenkann. Für eine Gruppenarbeit ist es auch gut da alle am gleichen arbeitenkönnen und man nur noch die Änderungen hochladen muss.
